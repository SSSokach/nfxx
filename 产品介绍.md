# AI办公助手 - 产品介绍

## 一、产品定位

**AI办公助手**是一款面向企业团队的智能协作平台，将即时通讯、文件管理、Excel在线编辑、AI智能助手、文件收集和表格追踪融为一体。系统基于"消息驱动 + AI赋能"的设计理念，让团队协作从"被动记录"升级为"主动追踪"。

**核心价值**：让AI不只是聊天，而是真正理解业务、追踪进度、驱动行动。

---

## 二、技术架构

| 层级 | 技术选型 |
|---|---|
| 前端 | Vue 3 + Vite + Axios |
| 后端 | FastAPI + SQLAlchemy |
| 数据库 | SQLite（单文件部署，零配置） |
| AI引擎 | GLM 大模型（支持 Function Calling） |
| 文件处理 | openpyxl（Excel解析）、marked（Markdown渲染） |

**系统特点**：单文件数据库，开箱即用；11个后端路由模块，60+个API端点；21张数据表覆盖完整业务域。

## 三、核心功能模块

### 模块1：即时通讯（IM）

**定位**：企业内部的"企业微信"，支持群聊和私聊。

**功能亮点**：
- 群聊与私聊切换，支持5人群聊、1对1私聊
- 消息类型：文本、文件、@所有人、@某人、回复引用
- 右键上下文菜单：复制、回复、转发、收藏、AI对话、多选
- 多选模式：批量复制、批量转发、批量发起AI对话

**演示数据**：
- 全员周报群：含@所有人消息"周五下午2点开项目评审会议"
- 项目组：含回复引用消息"我来统一接口返回格式"
- 张三与李四私聊：含工作约定"明天上午10点前把接口文档发给我"

**切换视角**：左上角下拉切换用户，可体验不同角色的消息视图。

---

### 模块2：文件管理与在线预览

**定位**：团队文件共享中心，支持在线预览和编辑。

**功能亮点**：
- **Markdown在线预览**：支持标题、列表、代码块、任务列表渲染
- **Excel在线编辑**：直接在浏览器中编辑Excel单元格
- 文件上传：支持拖拽上传和手动输入两种方式
- 文件下载：支持原始文件下载

**演示数据**：
- `项目计划.md`：含任务列表、里程碑追踪
- `周报填写模板.xlsx`：5人行，3人已填、2人未填

**Excel编辑特色**：
- 单元格直接可编辑，无需下载
- 修改过的单元格以**黄色高亮**显示
- 关闭预览时自动保存
- 保存时自动联动更新文件收集任务和待办状态

---

### 模块3：AI智能助手

**定位**：不是简单的聊天机器人，而是理解业务上下文的智能助手。

**功能亮点**：
- **AI对话**：基于GLM大模型，支持Function Calling调用工具
- **智能回复**：根据聊天上下文，一键生成多条候选回复
- **消息意图分类**：AI自动识别消息类型
- **AI日报生成**：总结今日工作动态
- **待办优先级排序**：AI智能排序
- **会议纪要**：从群聊提取会议要点
- **文件摘要**：AI生成文件结构化摘要
- **工作报告**：基于待办状态自动生成

**Token用量管控**：
- 每用户每日Token用量统计
- 进度条可视化展示
- 超限自动限速

**演示入口**：
- 右侧AI面板→AI对话Tab：输入"帮我总结今天的工作"
- 聊天区→💡智能回复按钮：点击生成候选回复

---

### 模块4：智能待办系统

**定位**：让消息不只是被看到，而是被追踪、被执行。

**功能亮点**：

#### 4.1 候选待办（AI自动检测）
- 点击"AI扫描"按钮，系统自动扫描群聊消息
- 智能识别潜在的待办事项（如截止日期、任务分配）
- 检测到Excel文件收集请求，自动生成候选
- **确认按钮**：将候选转为正式待办
- **忽略按钮**：放弃不重要的待办

#### 4.2 @所有人消息摘要
- 群聊中的@所有人消息自动生成摘要
- 可一键将摘要转为待办

#### 4.3 @某人消息提取
- 群聊中@张三的消息，自动为张三生成待办
- 私聊中的任务约定，自动提取为待办

#### 4.4 待办状态管理
- **pending**：待处理
- **completed**：已完成
- **deleted**：已删除（可恢复）
- **overdue**：逾期（deadline已过）

**演示数据**：
- 候选待办2条：1条pending（周五前提交周报）+ 1条dismissed
- 正式待办4条：含pending、completed、文件收集待办
- @所有人摘要1条

---

### 模块5：文件收集（填表催办）

**定位**：告别"群发催收Excel"的低效模式。

**功能亮点**：
- **自动检测**：群聊中的Excel文件，点击"📋生成待办"按钮
- **AI解析模板**：读取Excel表头，自动识别待填写人员名单
- **双视角待办**：
  - 发起人看到：进度跟踪待办"等待2人提交（3/5）"
  - 填写人看到：填写待办"[填写文件] 周报填写模板.xlsx"
- **进度条可视化**：紫色进度条显示填写比例
- **未填写人列表**：以红色标签列出未提交人员
- **Excel扫描**：点击"扫描进度"按钮，重新读取Excel填写状态
- **自动联动**：填写人提交后，发起人的待办进度自动更新

**演示数据**：
- 发起人张三的待办：`[文件收集] 周报填写模板.xlsx - 等待2人提交（3/5）`
- 填写人郑十的待办：`[填写文件] 周报填写模板.xlsx - 由 张三 发起`
- FileTracker：未填人员 = 赵六,郑十

**业务闭环**：
1. 张三在群里发Excel → 2. 系统检测生成待办 → 3. 群成员填写Excel → 4. Excel保存触发进度更新 → 5. 发起人看到实时进度 → 6. 未填人员自动高亮催办

---

### 模块6：在线表格

**定位**：比Excel文件更轻量的多人协作填写方案。

**功能亮点**：
- **在线创建**：自定义列结构（如本周工作、进度、下周计划）
- **自动分配**：为每个指定成员生成空行
- **行级编辑**：每人只填自己的行
- **状态追踪**：实时显示已填/未填比例
- **关闭表格**：支持手动关闭表格

**演示数据**：
- 在线表单"7月工作汇报"：5人，3人已填、2人未填
- FormTracker已关联此表单，支持"查看表格"按钮

**与文件收集的区别**：
| 维度 | 文件收集 | 在线表格 |
|---|---|---|
| 载体 | Excel物理文件 | 数据库行数据 |
| 编辑方式 | Excel在线编辑器 | 行内表单 |
| 适用场景 | 复杂表格、公式 | 简单信息收集 |

---

### 模块7：表格追踪

**定位**：跨系统的填写进度追踪器。

**功能亮点**：
- **创建追踪**：指定表单URL和待填写人员
- **一键检测**：批量检测所有追踪中的表格
- **@催办**：在群聊中自动@未填写人员
- **取消追踪**：任务完成后关闭

**演示数据**：
- 追踪"7月工作汇报"：3/5已填，关联到在线表单
- 点击"检测"按钮刷新进度
- 点击"催办"按钮在群里@未填人员

### 模块8：邮件系统与回复追踪

**定位**：邮件不只是收发，更是跨部门协作追踪。

**功能亮点**：

**演示数据**：

- 技术部已回复，产品部和设计部未回复
- EmailTracker状态：tracking

## 六、技术亮点

### 1. 消息驱动的待办生成
传统OA系统需要手动创建任务，本系统从聊天消息中自动识别待办，让"说过的话"变成"要做的事"。

### 2. Excel智能解析
系统不只是存储Excel文件，而是能读取表头、识别人员名单、追踪填写状态，实现真正的"文件收集自动化"。

### 3. AI Function Calling
AI不只是聊天，可以调用工具：扫描消息、生成摘要、创建待办、生成报告。AI成为业务流程的参与者。

### 4. 跨模块数据联动
Excel保存 → 更新文件收集任务 → 更新待办进度 → 更新发起人视图。一次操作，多处同步。

### 5. 邮件部门级追踪
不只是追踪"是否回复"，而是按部门记录回复状态，让跨部门协作可见可控。
